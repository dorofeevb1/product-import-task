# Тестовое задание — Импорт/Экспорт товаров

> **Результат:** заполненная база данных через **Slim + Angular** с использованием `.xlsx` файла.

## Стек

| Слой | Технологии |
|------|-----------|
| Backend | Slim + Doctrine ORM |
| Frontend | Angular / Vite (TypeScript) |
| Хранилище | Реляционная БД (MySQL / PostgreSQL) |

Проект необходимо выложить на **GitHub / GitLab** (приватный репозиторий на время разработки, открытый — на момент сдачи).
Файлы-примеры импорта `.xlsx` прикреплены.

## Страницы приложения

| # | Страница | Описание |
|---|----------|----------|
| 1 | **Импорт товаров** | Кнопка загрузки `.xlsx` файла + вызов API импорта |

## Задачи

### 1. База данных и миграции

#### Таблица `products`

| Поле | Описание |
|------|----------|
| `id` | Первичный ключ |
| `external_code` | Уникальный внешний код товара |
| `name` | Наименование |
| `description` | Описание |
| `price` | Цена продажи |
| `discount` | Скидка в % = `(Цена продажи − Закупочная цена) / Цена продажи × 100` |

#### Таблица `product_attributes`

- [ ] Поля `key` и `value` (формат ключ–значение)
- [ ] Источник: столбцы xlsx с префиксом `"Доп. поле"`
- [ ] Привязка к товару через внешний ключ

#### Таблица `product_images`

- [ ] Поля `url` (исходная ссылка) и `path` (локальный путь)
- [ ] При импорте изображения **скачиваются** по ссылке из файла
- [ ] Привязка к товару через внешний ключ

---

### 2. Функционал страниц

#### Страница импорта товаров

- [ ] Кнопка выбора и загрузки `.xlsx` файла
- [ ] Парсинг файла → запись в БД
- [ ] Скачивание изображений и привязка к товарам
- [ ] Асинхронный импорт через очередь — Symfony Messenger + RabbitMQ/Kafka (не синхронный запрос)
- [ ] Обработка ошибок импорта: невалидные строки не прерывают процесс, собираются в отчёт и возвращаются клиенту
- [ ] Повторный импорт: upsert по `external_code` — дубликаты не создаются

#### Страница списка товаров

- [ ] Загрузка товаров из API
- [ ] Список с кликабельными элементами → переход на карточку товара
- [ ] Серверная пагинация (`?page=&limit=`) — данные запрашиваются постранично, не client-side slice
- [ ] Поиск и фильтрация по полям (`name`, диапазон `price`)
- [ ] NgRx Store + Effects для списка товаров:
  - Actions: `loadProducts`, `loadProductsSuccess`, `loadProductsFailure`
  - Effect: запросы через `ProductService`
  - Selectors: `selectAllProducts`, `selectProductsLoading`, `selectTotalPages`
- [ ] Компонент использует `async pipe` и `store.select()`, не прямой `.subscribe()`

#### Карточка товара

- [ ] Все основные поля товара
- [ ] Дополнительные атрибуты (`product_attributes`)
- [ ] Изображения (`product_images`)
- [ ] Индикатор прогресса импорта — polling статуса задачи или WebSocket
- [ ] Отображение отчёта об ошибках после импорта

## Требования к архитектуре

### Backend

- [ ] Логика импорта (парсинг, скачивание изображений) — в отдельном Service-слое, не в контроллере
- [ ] Репозиторий-паттерн: запросы к БД через отдельные Repository-классы
- [ ] Контроллер только принимает запрос, делегирует, возвращает ответ
- [ ] Транзакция на запись товара + атрибутов + изображений (атомарность)
- [ ] Валидация входящего файла: тип MIME, размер, расширение

### Frontend

- [ ] Для каждой сущности — отдельный Angular-сервис (`ProductService`, `ImportService`) — не прямой `HttpClient` в компонентах
- [ ] TypeScript-интерфейсы для всех сущностей в `src/app/models/` (`Product`, `ProductAttribute`, `ProductImage`)
- [ ] Роутинг в отдельном `app.routes.ts` или модульном routing-файле
- [ ] `NgRx Store + Effects` хотя бы для списка товаров (см. требования выше)
- [ ] `strict: true` в `tsconfig.json`
